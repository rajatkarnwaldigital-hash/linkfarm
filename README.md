# link-farm-outreach

Identifies companies buying backlinks from known link farms and generates personalised cold email hooks referencing the specific farms found. Part of the EthicalSEO outbound engine.

---

## What it does

Takes a pre-cleaned list of domains extracted from link farm outgoing link exports (sourced from Ahrefs) and runs them through a 2-stage pipeline:

1. **Qualify** â€” filters down to mid-market B2B/SaaS companies worth contacting (active site, English, Authority Score 10â€“60, Claude ICP check). Bot-blocked sites are kept â€” a site blocking automated requests is provably active and is still your ICP.
2. **Generate hooks** â€” for each qualified domain, writes a 2-sentence personalised cold email opening that names the specific link farms found, the number of farms, and the ranking risk â€” using Claude Haiku

The signal: if a company's domain appears in the outgoing links of a known link farm, they bought those links. That's a concrete, verifiable proof point that writes the outreach for you.

---

## Input preparation

`link_farm_qualify.py` expects a CSV called `link_farm_domains.csv` with one row per domain, already tiered and scored. **This file is not built by hand â€” it's generated by `process_link_farms.py`**, which runs on the raw Ahrefs exports.

### Step 0 â€” `process_link_farms.py` (run this first)

**Input:** one CSV per known link farm, exported from Ahrefs (Outgoing Links â†’ Linked Domains), each with columns `Link Farm Domain` and `Target domain`. List the filenames in the `INPUT_FILES` array at the top of the script.

**What it does:**
1. Reads every input CSV and extracts the clean root domain (strips `https://`, `www.`, paths) for each linked-to target
2. Drops known noise domains (social platforms, app stores, big publishers, podcast hosts â€” see `NOISE_DOMAINS`)
3. Tracks, per domain, which farms link to it and how many distinct pages on each farm link to it
4. Filters out anything with fewer than 2 total linking pages (too weak a signal)
5. Assigns `tier`: `hot` if the domain appears across 2+ farms, `warm` if it's a single farm with 2+ pages
6. Sorts by tier, then farm_count, then total_pages (strongest prospects first)
7. Writes `link_farm_prospects.csv` with columns: `domain`, `tier`, `farm_count`, `total_pages`, `farms`, `farm_detail`

**Run it:**
```bash
python3 process_link_farms.py
```

**Known limitation:** the noise filter only blocks known low-value platforms â€” it does **not** catch large, high-authority sites (e.g. Statista, HubSpot, Investopedia) that link farms cite organically as references rather than sell links to. Those get caught downstream by `link_farm_qualify.py`'s Authority Score band (10â€“60), which filters out anything too big/authoritative to be a real link-buying prospect. Don't try to hardcode a brand blocklist here â€” the AS filter is the correct fix.

**Output â†’ input handoff:** rename/save `link_farm_prospects.csv` as `link_farm_domains.csv` (or update `INPUT_FILE` in `link_farm_qualify.py`) before running Step 1 below.

## Scripts

### 1. `link_farm_qualify.py`

Filters the domain list down to qualified prospects worth contacting.

**Input:** `link_farm_domains.csv`

**Output:** `link_farm_qualified.csv`

Output columns:
| Column | Description |
|--------|-------------|
| `domain` | Domain |
| `tier` | `hot` or `warm` â€” passed through from input |
| `farm_count` | Number of farms â€” passed through |
| `total_pages` | Total link farm pages â€” passed through |
| `farms` | Farm list â€” passed through |
| `farm_detail` | Farm detail â€” passed through |
| `site_alive` | `True` if site responded or is bot-blocked |
| `is_english` | `True` if English detected; `True` assumed for bot-blocked sites |
| `authority_score` | SEMrush Authority Score |
| `is_good_prospect` | `True` if Claude Haiku confirmed mid-market B2B/SaaS ICP |
| `qualified` | `True` if all filters passed |
| `fail_reason` | Why the domain was filtered out (if applicable) |

**Filters applied (in order):**
1. Site alive check â€” tries 4 URL variants (https/http Ă— www/non-www), HEAD then GET fallback. **Bot-blocked sites (403) are kept and passed through** â€” they are provably active
2. Language detection â€” reads `<html lang>` attribute. Skipped for bot-blocked sites (no response body); English assumed
3. Authority Score 10â€“60 band (SEMrush `backlinks_overview`). Below 10 = too small/spammy; above 60 = major brand, not a fit
4. ICP check â€” Claude Haiku YES/NO: is this a real mid-market B2B company or SaaS product that would care about SEO rankings? Rejects major brands, news/media sites, directories, affiliates, non-profits, and social platforms

**SEMrush cost:** ~10 units per domain (single `backlinks_overview` call)

**Performance:** 10 parallel workers via `ThreadPoolExecutor`. Checkpoints every 100 rows to `link_farm_qualified.csv.checkpoint` â€” safe to interrupt and resume.

### 2. `link_farm_hook_generator.py`

Generates a 2-sentence personalised cold email hook for each qualified domain.

**Input:** `link_farm_qualified.csv` â€” all rows, no filter on `qualified` column applied internally; feed it only qualified rows (the output of `link_farm_qualify.py` already contains only qualified domains if you filter before passing, or pass the full file and it will process all rows)

> **Tip:** To process only qualified domains, filter `link_farm_qualified.csv` to rows where `qualified == True` before running, or pass the full file â€” the script processes every row it receives.

**Output:** `link_farm_hooks.csv`

Output columns:
| Column | Description |
|--------|-------------|
| `domain` | Domain |
| `tier` | `hot` or `warm` |
| `farm_count` | Number of distinct link farms |
| `total_pages` | Total link farm pages linking to this domain |
| `farms` | Farm list |
| `farm_detail` | Farm detail |
| `authority_score` | Passed through from qualify output |
| `hook` | 2-sentence personalised cold email opening |
| `hook_status` | `ok` or `api_error` |

**Hook generation logic (single Claude Haiku call per domain):**

1. Picks the top 2 most prominent farm names by page count from `farm_detail`
2. Selects hook framing based on tier:
   - **Hot** (`farm_count` â‰Ą 2): leads with number of farms â€” "found across X different link farms including Farm1 and Farm2"
   - **Warm** (single farm): leads with page volume â€” "found on Farm1 and Farm2 across X different pages"
3. Sentence 1 â€” states what was found, names the farms, sounds like a manual backlink review
4. Sentence 2 â€” states the ranking risk factually, no alarmism

Hook rules enforced in the prompt: no buzzwords (toxic, link profile, leverage, synergy), no em dashes, no mention of the agency, no CTA, plain text only, max 2 sentences.

**Performance:** Single-threaded (`THREADS = 1`) with 0.5s delay between calls â€” conservative to avoid Anthropic rate limits. On 500 domains expect ~5 minutes.

## Usage

### Step 1 â€” Prepare your input

Export outgoing domain reports from Ahrefs for each known link farm, merge, clean out non-website rows, and save as `link_farm_domains.csv`. See Input preparation section above for exact column requirements.

### Step 2 â€” Run link_farm_qualify.py

```bash
python link_farm_qualify.py
```

Produces `link_farm_qualified.csv`. On 2,000 domains expect ~25 minutes with 10 threads and ~20,000 SEMrush units.

### Step 3 â€” Filter and run link_farm_hook_generator.py

Filter `link_farm_qualified.csv` to rows where `qualified == True`, save as a separate file if preferred, then run:

```bash
python link_farm_hook_generator.py
```

Produces `link_farm_hooks.csv`. On 500 qualified domains expect ~5 minutes.

## Known Limitations

- Bot-blocked sites pass through without language verification. In practice, English can be inferred from the farm context (link farms typically target English-language SEO), but manually spot-check a sample if running on non-English geographies.
- The Claude ICP check uses domain name only â€” it cannot visit the site for bot-blocked domains. Accuracy is high for well-known SaaS brands but may misclassify niche or obscure domains.
- Hook quality depends on `farm_detail` being populated. If `farm_detail` is empty, the hook falls back to generic farm count framing without specific page numbers.
- The pipeline does not verify that the link farm pages still exist at run time â€” Ahrefs data may be weeks old. A small percentage of hooks may reference farms that have since been deindexed.

## Plusvibe Sequence (Reference)

Load `link_farm_hooks.csv` into Plusvibe. Map the `hook` column to `{{hook}}` in the email copy.

| Step | Day | Purpose |
|------|-----|---------|
| Email 1 | Day 1 | Hook-led opener â€” `{{hook}}` as the first paragraph |
| Email 2 | Day 4 | Follow-up â€” reinforce the risk angle, add social proof |

| Email 4 | Day 14 | Break-up email |

Tone note: copy for this sequence uses a less accusatory framing â€” leads with "we noticed" rather than "you have a problem". Give the prospect plausible deniability (old agency, previous contractor).
